Security Keys Required at Deployment

When to Generate Keys

Generate keys at deployment time on the production server, NOT before.

  • Development: Use placeholder keys (already in .env.example) - fine for local testing
  • Production: Generate fresh, unique keys on each server/environment

Why Generate at Deployment?

  • Different keys per environment - Staging and production should have different keys
  • Keys shouldn't be in git - If committed, they're exposed in repo history forever
  • Rotation capability - Generate at deployment allows easy key rotation without code changes

Generate Each Key On Server

Run this command 3 times on your production server:

python -c "import secrets; print(secrets.token_urlsafe(32))"
KeyPurposeCan Rotate?
SECRET_KEYApp session signingYes (users re-login)
JWT_SECRET_KEYJWT token signingYes (tokens invalidated)
PII_ENCRYPTION_KEYEncrypt sensitive dataNo*
Important: PII_ENCRYPTION_KEY

The PII_ENCRYPTION_KEY is special - once you encrypt data with it, you cannot change it without losing access to all encrypted PII (emails, names, etc.).

  • Generate it once at first deployment
  • Back it up securely (password manager, secrets vault)
  • Never commit it to git or share it

Critical: PII_ENCRYPTION_KEY Management

The PII_ENCRYPTION_KEY encrypts all PII (emails, names, etc.). It cannot be rotated without losing access to encrypted data.

  • Generate once at first deployment
  • Store in a secrets manager (AWS Secrets Manager, HashiCorp Vault)

  • Document key custodians and access procedures
Enterprise Best Practices
  • Use AWS Secrets Manager, HashiCorp Vault, or GCP Secret Manager
  • Rotate SECRET_KEY and JWT_SECRET_KEY quarterly
  • Audit key access logs regularly
  • Never store keys in environment files on disk - inject at runtime

Protecting Your Software from Copying & Theft

Multi-layered approach to protect your intellectual property.

Critical

1. License Key System

Implement a license validation system that phones home to verify legitimacy.

Services: Keygen.sh ($99/mo), Cryptlex, LemonSqueezy

Critical

2. Domain/IP Locking

Bind each license to specific domains or IP addresses.

High

3. Code Obfuscation

Make code difficult to read and reverse engineer.

Python: PyArmor ($59), Cython, Nuitka

High

4. Private Docker Registry

Never publish images publicly. Use AWS ECR, GCR, or self-hosted Harbor.

High

5. Heartbeat/Phone Home

Regular check-ins to your license server for validation.

Medium

6. Code Watermarking

Embed unique identifiers in each client's deployment for tracing leaks.

Security Implementation Checklist

  • Implement license key validation system
  • Set up private Docker registry
  • Add domain/IP locking to licenses
  • Obfuscate Python code with PyArmor or Cython
  • Implement heartbeat/phone-home system
  • Use secrets manager (not plain .env files)
  • Add unique watermarks per deployment
  • Create EULA and ToS documents
  • Set up monitoring for unauthorized deployments
