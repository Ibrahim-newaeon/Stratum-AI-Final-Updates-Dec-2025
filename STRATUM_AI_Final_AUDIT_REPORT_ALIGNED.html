1. Executive Overview

Core

Platform Description

Stratum AI is a Revenue Operating System with Trust-Gated Autopilot architecture. The platform provides unified marketing intelligence across multiple advertising platforms with ML-driven predictions, real-time analytics, and automated campaign optimization.

Core Concept

Signal Health Check -> Trust Gate -> Automation Decision | | | [HEALTHY] [PASS] [EXECUTE] [DEGRADED] [HOLD] [ALERT ONLY] [UNHEALTHY] [BLOCK] [MANUAL REQUIRED]

Key Differentiator

Automation executes ONLY when signal health passes safety thresholds. This protects advertisers from making costly mistakes based on unreliable data.

6. Trust Engine & Autopilot

Critical System

Signal Health Thresholds

Status Score Range Color Autopilot Mode
HEALTHY 70-100 Green Normal - Full autopilot enabled
RISK 60-69 Yellow Limited - Conservative actions only
DEGRADED 40-59 Orange Cuts Only - Pause/reduce only
CRITICAL 0-39 Red Frozen - No automation

Signal Health Components

EMQ Score (40% weight)

Event Match Quality from platform data. Measures how well conversion events match user data.

Freshness Score (25% weight)

How recent the data is. <24h = 100%, 24-48h = decay, >48h = 0%

Variance Score (20% weight)

Platform vs GA4 revenue variance. Warning at 10%, critical at 15%.

Anomaly Score (15% weight)

Z-score analysis on spend, conversions, CPA, ROAS. Threshold: z > 3.0

Action Risk Classification

Risk Level Required Score Actions
High Risk 80+ increase_budget, launch_new_campaigns, expand_targeting, increase_bid
Standard 70+ update_budget, update_bid, update_status, update_targeting
Conservative 60+ pause_underperforming, reduce_budget, reduce_bid
Always Allowed Any pause_all, emergency_stop
