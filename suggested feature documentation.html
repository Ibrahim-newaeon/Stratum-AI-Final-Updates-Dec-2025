Feature #1 · Dashboard UX

AI Morning Briefing

Completed
Overview

A personalized daily intelligence digest that appears at the top of the dashboard on load. Shows overnight changes, KPI snapshot, signal health status, and prioritized action items — replacing 15 minutes of manual clicking with one glanceable card.

Technical Implementation
Backend: GET /api/v1/dashboard/morning-briefing
File: backend/app/api/v1/endpoints/dashboard.py
  • Returns MorningBriefingResponse with:
  • Personalized greeting using user's first name
  • Executive summary narrative (2–3 sentences)
  • Portfolio health assessment (strong / steady / needs_attention / critical)
  • KPI snapshot: spend, revenue, ROAS, conversions with % changes
  • Signal health status and score
  • Top 5 overnight changes with human-readable narratives
  • Top 5 prioritized action items with impact estimates
  • Campaign counts, scale/fix candidates
  • Auth: Requires authenticated user (CurrentUserDep)
  • Data: Aggregates from Campaign table, AuditLog, signal health
Frontend Component: MorningBriefingCard.tsx
File: frontend/src/components/dashboard/MorningBriefingCard.tsx
  • Placement: First element on the Dashboard Overview page
  • Sun icon + personalized greeting header
  • Portfolio health badge (color-coded)
  • 4 KPI mini-cards with trend indicators
  • Signal health bar with autopilot status
  • Collapsible "Overnight Changes" section with directional icons
  • Collapsible "Actions Today" section with priority coloring
  • Session-dismissible (X button)
  • Design: Light theme, bg-card, border, no glassmorphism, Clash Display headings
API Integration: dashboard.ts
File: frontend/src/api/dashboard.ts
  • Types: MorningBriefingResponse, BriefingChangeItem, BriefingActionItem
  • Hook: useMorningBriefing() — 10min stale, 30min refetch
  • API function: dashboardApi.getMorningBriefing()
Data Flow
User opens dashboard → Overview.tsx renders <MorningBriefingCard /> → useMorningBriefing() calls GET /dashboard/morning-briefing → Backend queries Campaign table for current metrics → Computes day-over-day changes → Generates narrative from changes + signal health → Returns structured briefing → Card renders greeting, KPIs, changes, actions → User can dismiss or collapse sections

Feature #5: Predictive Budget Autopilot
Confidence-scored budget reallocation with trust gate validation and forecast projections
Architecture Overview
The Predictive Budget Autopilot analyzes all campaigns to generate confidence-scored budget reallocation recommendations. It uses composite scoring (ROAS delta 60%, CPA delta 40%), validates through the trust gate, and projects forecast outcomes if recommendations are applied.
Data Flow
Campaign Metrics (spend, revenue, conversions)
        ↓
Portfolio Averages (avg spend, revenue, conversions)
        ↓
Campaign Scoring (_calculate_campaign_score)
  • ROAS delta (60% weight)
  • CPA delta (40% weight)
  • Confidence from volume + spend + score magnitude
        ↓
Action Classification (_determine_action)
  • scale: score ≥ 0.25, confidence ≥ 0.5
  • reduce: score ≤ -0.25, confidence ≥ 0.4
  • pause: score ≤ -0.25 AND ROAS < 0.5
  • maintain: default
        ↓
Budget Calculation (_calculate_recommended_spend)
  • Scale: +15-30% (modulated by confidence)
  • Reduce: -10-20% (modulated by confidence)
  • Pause: $0
        ↓
Risk Analysis (_identify_risk_factors)
  • Low conversion volume (<10)
  • Below confidence threshold (<0.6)
  • ROAS below breakeven (<1.0)
  • Low spend learning phase (<$50)
        ↓
Trust Gate Validation
  • PASS: signal health ≥ 70 → autopilot eligible
  • HOLD: signal health ≥ 40 → advisory only
  • BLOCK: signal health < 40 → changes suspended
        ↓
Forecast Projection
  • Projected spend, revenue, ROAS, conversions
  • Change percentages vs current
  • Confidence level (high/medium/low)
Backend Models
ModelPurpose
CampaignBudgetInsightPer-campaign recommendation with action, confidence, metrics, risk factors
BudgetForecastProjected spend/revenue/ROAS/conversions if recommendations applied
PredictiveBudgetResponseFull response: trust gate, recommendations, forecast, summary
Frontend Components
ComponentPurpose
PredictiveBudgetCardMain card: trust gate badge, stats bar, forecast, recommendations
ForecastCardProjected impact: spend, revenue, ROAS, conversions with change indicators
RecommendationRowExpandable per-campaign row: action, confidence, metrics, risk factors

AI-Generated Reports automatically produce executive-quality performance reports by aggregating campaign metrics, analyzing platform breakdowns, identifying top/bottom performers, generating narrative insights, and producing actionable recommendations. Data Flow
Campaign Metrics (all campaigns, all platforms)
        ↓
Aggregation (total spend, revenue, conversions, ROAS, CPA)
        ↓
Health Grading (_calculate_grade)
  • A: ROAS ≥ 4.0 + 100+ conversions
  • B: ROAS ≥ 3.0 + 50+ conversions
  • C: ROAS ≥ 2.0 + 20+ conversions
  • D: ROAS ≥ 1.0
  • F: Below breakeven
        ↓
Platform Analysis (_analyze_platforms)
  • Per-platform spend, revenue, ROAS, conversion counts
  • Spend share percentage
  • Narrative commentary per platform
        ↓
Campaign Highlights (_find_highlights)

  • Bottom 3 by ROAS with risk narrative
        ↓
Insight Generation (_generate_insights)

  • Low conversion warnings
  • Portfolio ROAS milestones

Recommendations (_generate_recommendations)
  • Scale top performers

Feature #8: Unified Notifications with AI Priority
AI-scored notification feed aggregating campaigns, signals, pacing, and system alerts
Architecture Overview
Unified Notifications aggregates alerts from multiple sources (campaign performance, signal health, system events, existing DB notifications), scores each by urgency, impact, and actionability using a weighted formula, then returns a priority-ranked feed with suggested actions and category grouping.
Priority Scoring Formula
Priority = (Urgency × 0.5) + (Impact × 0.3) + (Actionability × 0.2)

Urgency (0-100):
  • Severity base: critical=90, error=75, warning=50, info=20
  • Recency boost: <1hr +10, <6hr +5, >24hr -10
  • Time-sensitivity bonus: +15

Impact (0-100):
  • Spend at risk: ≥$10K=40, ≥$5K=30, ≥$1K=20
  • Deviation %: ≥50%=30, ≥25%=20, ≥10%=10
  • Campaigns affected: ≥5=20, ≥2=10

Actionability (0-100):
  • Auto-resolvable=90, suggested action=70, manual=30

Priority Labels:
  • Critical: ≥75  |  High: ≥50  |  Medium: ≥30  |  Low: <30
Notification Sources
SourceTriggers
CampaignCritical ROAS (<0.5x), zero conversions, top performer (5x+ ROAS)
Signal HealthHealth critical (<40), health degraded (<70)
SystemNo active campaigns, portfolio below breakeven
