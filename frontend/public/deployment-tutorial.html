Deployment Steps

1

Environment Setup

Required

Clone the repository and configure environment variables.

# Clone the repository
git clone https://github.com/your-org/stratum-ai.git
cd stratum-ai

# Copy environment template
cp .env.example .env

# Edit environment variables
nano .env

Required Environment Variables 

# Database
DATABASE_URL="postgresql://user:pass@localhost:5432/stratum"
REDIS_URL="redis://localhost:6379"

# Security
SECRET_KEY="your-super-secret-key-min-32-chars"
JWT_SECRET="your-jwt-secret-key"

# API Keys
STRATUM_API_KEY_LIVE="strat_live_..."
STRATUM_API_KEY_TEST="strat_test_..."
2

Database Configuration

Required

Initialize PostgreSQL and run migrations.

# Start database services
docker compose up -d postgres redis

# Run database migrations
make migrate

# Seed initial data (optional)
python -m scripts.seed_data
PostgreSQL

Primary data store for profiles, segments, and automations.

Redis

Caching, session storage, and Celery task queues.

Trust Engine Configuration

Core Feature

Configure the Trust-Gated Autopilot thresholds.

Trust Gate Thresholds

BLOCK HOLD PASS
0-39
Manual Required
40-69
Alert Only
70-100
Autopilot Active
# config/trust_engine.yaml
trust_engine:
  thresholds:
    healthy: 70      # PASS - autopilot enabled
    degraded: 40     # HOLD - alerts only

  signals:
    - name: "data_freshness"
      weight: 0.3
    - name: "source_reliability"
      weight: 0.25
    - name: "anomaly_score"
      weight: 0.25
    - name: "completeness"
      weight: 0.2

Webhook Configuration

Recommended

Event Description Use Case
