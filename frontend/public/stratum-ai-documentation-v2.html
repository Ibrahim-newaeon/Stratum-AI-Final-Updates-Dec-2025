Platform Architecture

In this section

System Overview

Stratum AI is a Revenue Operating System with Trust-Gated Autopilot architecture. Automation executes ONLY when signal health passes safety thresholds. The core flow is: Signal Health Check → Trust Gate → Automation Decision.

Signal Sources → Signal Health Check → Trust Gate → Automation Decision [HEALTHY 70-100] → [PASS] → [EXECUTE] [DEGRADED 40-69] → [HOLD] → [ALERT ONLY] [UNHEALTHY 0-39] → [BLOCK] → [MANUAL REQUIRED]
The Trust Gate is the central safety mechanism. No automation can bypass the trust evaluation, ensuring data-driven decision-making at every step.

Trust & Signal Engine

In this section

Trust Engine Overview

The Trust Engine is the core decision-making component that evaluates signal health before allowing automation execution. It consists of four key components working in sequence.

[Signal Sources] → [Collectors] → [Health Calculator] ↓ [Trust Gate] ↓          ↓ [PASS]    [HOLD/BLOCK] ↓          ↓ [Execute] [Alert/Manual]

Signal Collectors

Gather data from ad platforms, analytics, CRM, and webhooks into a unified signal stream.

Health Calculator

Compute composite reliability scores from weighted signal components.

Trust Gate

Binary decision point: pass, hold, or block automation based on health score.

Automation Executor

Execute approved actions: budget adjustments, bidding changes, campaign toggles.

Signal Health Score

A composite score from 0 to 100 measuring overall signal reliability. The score is computed from five weighted components.

ComponentWeightDescription
EMQ (Event Match Quality)35%How well tracked events match platform data
API Health25%API response times, error rates, availability
Event Loss20%Percentage of events lost in transit
Platform Stability10%Ad platform uptime and consistency
Data Quality10%Completeness and accuracy of data
70–100 Healthy (Green): Autopilot enabled, automation proceeds.
40–69 Degraded (Yellow): Alert and hold, no auto-execution.
0–39 Unhealthy (Red): Manual intervention required.

Trust Gate

The decision checkpoint that evaluates signal health before any automation can execute. Three enforcement modes control how strictly the gate is applied.

DecisionScoreAction
PASS≥ 70Automation proceeds normally
HOLD40–69Alert only, no execution
BLOCK< 40Requires manual approval

Enforcement Modes: Advisory (log only), Soft-Block (hold + alert), Hard-Block (full block).

stratum/core/trust_gate.pytrust gate UI indicators

A score from 0 to 100 measuring how well tracked events match with platform data. Higher EMQ indicates better attribution accuracy and more reliable signal health.

Customer Data Platform

In this section

CDP Overview

Unified customer records combining data from multiple sources. Each profile contains identity information, behavioral events, computed traits, and segment memberships.

The CDP is the single source of truth for customer data across all connected platforms and touchpoints.
models/cdp.py

Identity Resolution

The process of connecting disparate identifiers to a single customer profile. Maps anonymous visitors through to known customers via progressive identification.

Anonymous (device/session) ↓ Known (via email/login) ↓ Customer (via purchase)

The Identity Graph provides a visual representation showing how different identifiers (email, phone, device IDs) connect to form a unified profile.

Segments

Dynamic groups of profiles matching specific criteria. Segments update automatically as customer data changes.

TypeDescriptionExample
BehavioralBased on event patternsUsers who viewed product 3+ times
DemographicBased on profile attributesUsers in tier "Enterprise"
PredictiveBased on ML modelsHigh churn risk (score > 0.7)

Computed Traits

Derived attributes calculated from event history. These are automatically computed and updated as new events arrive.

  • Total purchases
  • Days since last activity
  • Average order value
  • Lifetime value (LTV)

RFM Analysis

Customer segmentation based on Recency (how recently they purchased), Frequency (how often they purchase), and Monetary value (how much they spend). Produces 12 distinct segments.

SegmentDescription
ChampionsBest customers: recent, frequent, high spend
Loyal CustomersConsistent buyers with strong frequency
Potential LoyalistsRecent buyers with growing frequency
New CustomersFirst-time recent buyers
PromisingRecent shoppers with average spend
Need AttentionAbove-average but declining activity
About to SleepBelow-average recency and frequency
At RiskPreviously valuable, now declining
Can't LoseHigh-value customers showing churn signals
HibernatingLow activity across all dimensions
LostNo recent activity, low value
OthersRemaining profiles not matching above

Lifecycle Stages

Customer journey positions tracked from first touch through to churn, enabling stage-appropriate engagement strategies.

Anonymous → Known → Lead → Customer → Active → At Risk → Churned

In this section

Authentication Data Encryption GDPR Compliance CCPA Compliance PII Protection Access Control

Authentication

Multi-layered authentication combining JWT tokens with OAuth2 and optional MFA via TOTP. Session management includes automatic token refresh and secure logout.

  • JWT tokens with short expiration + refresh flow
  • OAuth2 for third-party platform authentication
  • MFA via TOTP (Time-based One-Time Password)
  • Secure session management with automatic cleanup

Data Encryption

Multiple encryption layers protect data at rest and in transit.

LayerMethodPurpose
PII at restFernet encryptionProtect personal data in database
Platform syncSHA-256 hashingHash emails/phones for audience matching
Data in transitTLS 1.2+Encrypt all network communication
Tokenssecrets moduleCryptographically secure token generation
Comparisonshmac.compare_digest()Constant-time comparison to prevent timing attacks

GDPR Compliance

Full GDPR compliance with data export, right to erasure, and consent management features.

  • Data export in machine-readable format
  • Right to erasure (RTBF) with complete data deletion
  • Consent management with granular categories
  • Data processing records

CCPA Compliance

California Consumer Privacy Act compliance ensuring right to know, right to delete, and right to opt-out of data sale for California residents.

PII Protection

Personally identifiable information (email, phone, names) is encrypted with Fernet before storage. PII is never included in JWT claims and is hashed with SHA-256 for audience matching.

Rule: PII must never appear in logs, JWT tokens, or client-side storage. Always use encrypted database fields.

Access Control

Role-based permissions with tenant isolation, row-level security, and comprehensive audit logging of all access events.

  • Role-based access control (RBAC)
  • Tenant isolation at query level
  • Row-level security in PostgreSQL
  • Audit logging for all state-changing operations

Glossary

Key terms and definitions used throughout the Stratum AI platform

In this section

Core Concepts CDP Terms Advertising Platform Technical Monitoring Compliance

Core Concepts

Signal
An input data point collected from various sources including metrics, events, and webhooks. Signals are the foundation of the Trust Engine.
Signal Health
A composite score (0-100) measuring the reliability and quality of signals. Used by the Trust Gate to determine if automation should proceed.
Trust Gate
A decision checkpoint that evaluates signal health before allowing automation to execute. Can PASS, HOLD, or BLOCK.
Trust Engine
The core decision-making component consisting of Signal Collectors, Health Calculator, Trust Gate Evaluator, and Automation Executor.
Autopilot
Automated actions that execute when trust gate conditions are met: adjust budgets, modify bidding, pause/enable campaigns, scale spend.
EMQ (Event Match Quality)
A score (0-100) measuring how well tracked events match with platform data. Higher EMQ = better attribution accuracy.

CDP Terms

Profile
A unified customer record combining data from multiple sources: identity, behavioral events, computed traits, segment memberships.
Identity Graph
A visual representation showing how different identifiers (email, phone, device IDs) connect to form a unified profile.
Identity Resolution
The process of connecting disparate identifiers to a single customer profile: Anonymous to Known to Customer.
Segment
A dynamic group of profiles matching specific criteria: Behavioral (event patterns), Demographic (attributes), Predictive (ML models).
Computed Traits
Derived attributes calculated from event history: total purchases, days since last activity, average order value, LTV.
RFM Analysis
Customer segmentation based on Recency, Frequency, and Monetary value.
Lifecycle Stage
Customer journey position: Anonymous, Known, Lead, Customer, Active, At Risk, Churned.
Audience Sync
Push CDP segments to ad platforms: Meta Custom Audiences, Google Customer Match, TikTok/Snapchat Audiences.

Advertising

CAPI (Conversions API)
Server-side event tracking that sends conversion data directly to ad platforms, bypassing browser limitations.
Attribution Model
Method for assigning credit to marketing touchpoints: last-click, first-click, linear, time-decay, data-driven.
DDA (Data-Driven Attribution)
Machine learning approach to attribution using historical data to determine credit distribution.
ROAS
Return on Ad Spend = Revenue / Ad Spend.
MER
Marketing Efficiency Ratio = Total Revenue / Total Marketing Spend.
CAC
Customer Acquisition Cost = Total Marketing Spend / New Customers Acquired.
LTV
Lifetime Value = Average Order Value x Purchase Frequency x Customer Lifespan.

Platform

Tenant
An isolated customer account with separate database schema, user management, configurations, and analytics.
Multi-tenancy
Architecture pattern where a single instance serves multiple customers with complete data isolation.
Feature Flag
Configuration toggles that enable/disable functionality per tenant or globally.
Subscription Tier
Account level determining available features: Starter, Professional, Enterprise.

Technical

Celery
Distributed task queue for background job processing: data sync, report generation, scheduled tasks.
Redis
In-memory data store used for caching, Celery message broker, real-time pub/sub, and rate limiting.
WebSocket
Bidirectional communication protocol for real-time updates: EMQ changes, incidents, autopilot status.
SSE (Server-Sent Events)
Unidirectional server-to-client streaming for dashboard updates.
OAuth
Authentication protocol for connecting to ad platforms (Meta, Google, TikTok, Snapchat).
JWT (JSON Web Token)
Token-based authentication containing User ID, Tenant ID, Permissions, Expiration.
MFA
Multi-Factor Authentication via TOTP (Time-based One-Time Password).

Monitoring

Prometheus
Metrics collection system for request latency, error rates, queue depths, system health.
Grafana
Visualization platform for Prometheus metrics dashboards.
Sentry
Error tracking and performance monitoring service with stack traces.
Structured Logging
JSON-formatted logs with consistent fields: Request ID, Tenant ID, User ID, Timestamp, Event type.

Compliance

GDPR
General Data Protection Regulation (EU privacy law). Supports data export, right to erasure, consent management.
CCPA
California Consumer Privacy Act. Right to know, right to delete, right to opt-out.
PII
Personally Identifiable Information: email addresses, phone numbers, names, physical addresses.
Data Hashing
One-way encryption of PII using SHA-256 for platform sync and audience matching without exposing raw data.
